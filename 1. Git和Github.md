3. $ git reset --hard HEAD^    【或者版本号】
	`--hard`会回退到上个版本的已提交状态，而`--soft`会回退到上个版本的未提交状态，`--mixed`会回退到上个版本未添加但未提交的状态

## 命令
### ls
1. **-l**：以长格式显示文件和目录信息。每行显示一个文件或目录的详细信息，包括文件类型与权限、硬链接数、所有者、所属组、文件大小、修改时间以及文件名。例如：

  

bash

```bash
-rw-r--r--  1 user  staff  1024 Jan  1 10:00 file.txt
drwxr-xr-x  2 user  staff   64  Jan  1 10:00 dir
```

  

其中 `-rw-r--r--` 表示文件权限，`-` 代表普通文件（`d` 代表目录），后面每三个字符一组，分别对应所有者、所属组、其他用户的读（`r`）、写（`w`）、执行（`x`）权限 ；`1` 是硬链接数 ；`user` 是所有者 ；`staff` 是所属组 ；`1024` 是文件大小（单位字节）；`Jan 1 10:00` 是修改时间；`file.txt` 是文件名。  
2. **-a**：显示所有文件和目录，包括以 `.` 开头的隐藏文件和目录。在类 Unix 系统中，以 `.` 开头的文件或目录默认是隐藏的，使用该选项可以将它们显示出来。  
3. **-t**：按照文件的修改时间排序，最新修改的文件排在最前面。比如在你提供的图片中，`ls -ltr` 里的 `-t` 就是用于按时间排序，`-r` （下面会介绍）使得排序顺序反转，即最早修改的文件排在前面。  
4. **-r**：反转排序顺序。比如和 `-t` 结合使用，`-tr` 可以让最早修改的文件排在前面；和按文件名排序结合使用，就可以让文件名按逆序排列。  
5. **-h**：与 `-l` 选项一起使用时，以人类可读的格式显示文件大小，例如将字节数转换为 KB、MB、GB 等。比如 `1048576` 字节会显示为 `1.0M`。  
6. **-d**：当参数是目录时，仅列出目录本身的属性，而不是目录中的内容。例如 `ls -ld /home/user` 会显示 `/home/user` 目录本身的详细属性，而不是显示该目录下的文件和子目录。  
7. **-R**：递归列出目录及其子目录下的所有内容。如果目录 `test` 下还有子目录 `subtest` ，使用 `ls -R test` 会依次列出 `test` 目录下的内容，以及 `subtest` 目录下的内容。


# 远程仓库

你在本地有仓库别名，和github的仓库名可以不相同，本地只用别名，地址对了就行

**远程到本地**：git clone SSH地址
**推送**git push -u 《远程仓库别名》 mian本地:main远程   关联起来。
**拉取**git pull 远仓别名 远仓分支名:本地分名（会有冲突）
**查看** git remote -v（verbose详尽
**加入远仓**（该文件夹下） git remote add <远程仓库别名称>  SSH地址
之后，采用**拉取**命令
**删除远仓** git remote rm 《仓库别名》
